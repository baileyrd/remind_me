   Replace the sequential loop (lines ~372-385):
   ```python
   # BEFORE (remove):
   results: list[dict[str, Any]] = []
   for f in sorted(files):
       try:
           r = import_chat_file(
               file_path=str(f),
               category=category,
               tags=tags,
               extract_mode=extract_mode,
               max_length=max_length,
           )
           results.append(r)
       except (json.JSONDecodeError, UnicodeDecodeError, FileNotFoundError, OSError) as e:
           log.warning("Failed to import %s: %s", f.name, e)
           results.append({"status": "error", "file": f.name, "error": str(e)})
   ```

   With the concurrent version:
   ```python
   # AFTER:
   sem = asyncio.Semaphore(IMPORT_CONCURRENCY)

   async def _import_one(f: Path) -> dict[str, Any]:
       async with sem:
           try:
               return await asyncio.to_thread(
                   import_chat_file,
                   file_path=str(f),
                   category=category,
                   tags=tags,
                   extract_mode=extract_mode,
                   max_length=max_length,
               )
           except (json.JSONDecodeError, UnicodeDecodeError, FileNotFoundError, OSError) as e:
               log.warning("Failed to import %s: %s", f.name, e)
               return {"status": "error", "file": f.name, "error": str(e)}

   results = list(await asyncio.gather(*[_import_one(f) for f in sorted(files)]))
   ```

   The rest of the function (summary aggregation from `ok = [r for r in results...]` onward) stays unchanged.

Key details:
- `asyncio.Semaphore` MUST be created inside the async function body (not at module level) — creating it outside a running event loop raises `RuntimeError` on Python 3.10+
- `IMPORT_CONCURRENCY` is the module-level integer constant; the `sem` Semaphore instance is local
- `import_chat_file` stays synchronous — dispatch via `asyncio.to_thread` for thread pool execution
- `asyncio.gather` preserves order (results correspond to sorted files)
- `list()` wrap on `gather` result converts tuple to list for type consistency

4. Update `__all__` if `IMPORT_CONCURRENCY` should be exported (it should not — keep it internal). No change needed.

  <behavior>
    - RRF scoring: Given 3 memories ranked [A,B,C] by keyword and [B,A,C] by semantic, RRF(A) = 1/(60+1) + 1/(60+2) = ~0.0326, RRF(B) = 1/(60+2) + 1/(60+1) = ~0.0326, RRF(C) = 1/(60+3) + 1/(60+3) = ~0.0317. A and B tie, C is lower.
    - RRF with recency: Given two memories with equal keyword+semantic ranks but different created_at dates, the newer one gets a higher final RRF score.
    - Token budget: Given 3 memories with content lengths 400, 800, 1200 chars (~100, 200, 300 tokens), budget=250, returns first 2 (tokens_used=200) and trimmed=1.
    - Token budget default: 800 tokens when not specified.
    - Envelope: Every result includes total_candidates, returned, trimmed, tokens_used, budget.
    - RRF k configurable: REMIND_ME_RRF_K env var overrides default k=60.
    - Empty input: No candidates returns envelope with all zeros.
    - Keyword-only: When semantic results are empty, RRF uses keyword + recency only.
    - Semantic-only: When keyword results are empty (FTS syntax error), RRF uses semantic + recency only.
  </behavior>

    2. Create `remind_me_mcp/retrieval.py` with these components:

    **Config at module top:**
    ```python
    import os
    RRF_K: int = int(os.environ.get("REMIND_ME_RRF_K", "60"))
    ```

    **`rank_rrf(keyword_results, semantic_results, k=None) -> list[dict]`**
    - Takes two lists of memory dicts (keyword-ranked and semantic-ranked, both already in rank order).
    - Computes 3 signal ranks per memory:
      - keyword_rank: position in keyword_results (1-indexed), or len(keyword_results)+1 if absent
      - semantic_rank: position in semantic_results (1-indexed), or len(semantic_results)+1 if absent
      - recency_rank: all unique memories sorted by created_at DESC, position is rank (1-indexed)
    - RRF score per memory: sum(1/(k + rank)) for each of the 3 signals
    - Returns all unique memories sorted by RRF score DESC (higher = better), with `_rrf_score`, `_keyword_rank`, `_semantic_rank`, `_recency_rank` attached to each dict.
    - k defaults to module-level RRF_K.

    **`apply_token_budget(ranked_memories, budget) -> SearchEnvelope`**
    - Iterates ranked memories, estimating tokens as `len(content) // 4`.
    - Accumulates until budget is reached (0 means unlimited -- return all).
    - Returns a SearchEnvelope dataclass/dict with: memories (list), total_candidates (int), returned (int), trimmed (int), tokens_used (int), budget (int).

    **`SearchEnvelope` (TypedDict or dataclass):**
    ```python
    class SearchEnvelope(TypedDict):
        memories: list[dict]
        total_candidates: int
        returned: int
        trimmed: int
        tokens_used: int
        budget: int
    ```

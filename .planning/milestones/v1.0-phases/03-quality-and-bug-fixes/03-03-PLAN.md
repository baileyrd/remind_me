**db.py — Singleton connection with WAL + busy_timeout:**

Convert `_get_db()` from creating a new connection per call to a lazy singleton:

```python
_db_connection: sqlite3.Connection | None = None

def _get_db() -> sqlite3.Connection:
    """Return the lazily initialized SQLite database singleton.

    The connection is configured with WAL journal mode for concurrent reader
    access, busy_timeout for graceful lock contention, and
    check_same_thread=False so it can be used from asyncio.to_thread workers.
    """
    global _db_connection
    if _db_connection is not None:
        return _db_connection

    db = sqlite3.connect(str(DB_PATH), timeout=10, check_same_thread=False)
    db.row_factory = sqlite3.Row
    db.execute("PRAGMA journal_mode=WAL")
    db.execute("PRAGMA busy_timeout=5000")
    db.execute("PRAGMA foreign_keys=ON")

    # Load sqlite-vec extension if available
    try:
        import sqlite_vec
        db.enable_load_extension(True)
        sqlite_vec.load(db)
        db.enable_load_extension(False)
    except (ImportError, Exception) as e:
        log.debug("sqlite-vec not available: %s (vector search disabled)", e)

    _ensure_schema(db)
    _db_connection = db
    return db


def _close_db() -> None:
    """Close the singleton database connection. Called during shutdown."""
    global _db_connection
    if _db_connection is not None:
        _db_connection.close()
        _db_connection = None
```

Add `_close_db` to `__all__`.

Note: `check_same_thread=False` is required because async handlers may call `_get_db()` from different threads when using `asyncio.to_thread`. The sqlite3 module's thread check is overly conservative for WAL mode databases. We use a single connection (no concurrent writes from different threads) and SQLite's internal mutex handles the rest.

**tools.py — Wrap blocking calls with asyncio.to_thread:**

Add `import asyncio` at the top of tools.py.

In every tool handler that calls `_embed_and_store`, wrap the call with `await asyncio.to_thread(...)`:

1. `memory_add`: Replace `_embed_and_store(db, mem_id, params.content)` with `await asyncio.to_thread(_embed_and_store, db, mem_id, params.content)`

2. `memory_update`: Same wrapping for `_embed_and_store(db, params.memory_id, params.content)`

3. `remind_me_auto_capture`: Wrap both `_embed_and_store` calls:
   ```python

   await asyncio.to_thread(_embed_and_store, db, dialog_id, params.conversation[:2000])
   ```

4. `memory_search`: Wrap `_semantic_search` call:
   ```python
   sem_memories = await asyncio.to_thread(_semantic_search, db, params.query, params.limit)
   ```

5. `remind_me_reindex`: Wrap the embedding loop body using to_thread for the embedder call:

Do NOT wrap simple database reads/writes with asyncio.to_thread — only wrap the CPU-intensive embedding computations. SQLite reads and small writes are fast enough to run inline.

Add a `_migrate_schema(db: sqlite3.Connection) -> None` function that implements incremental schema migration using `PRAGMA user_version`:

1. Read current version: `db.execute("PRAGMA user_version").fetchone()[0]`
2. Apply migrations in order, each guarded by version check:

**Version 0 -> 1:**
- Add `capture_id` column to memories table: `ALTER TABLE memories ADD COLUMN capture_id TEXT DEFAULT NULL`
- Create index: `CREATE INDEX IF NOT EXISTS idx_memories_capture_id ON memories(capture_id)`
- Backfill capture_id from existing metadata JSON where present:
  ```sql
  UPDATE memories SET capture_id = json_extract(metadata, '$.capture_id')
  WHERE json_extract(metadata, '$.capture_id') IS NOT NULL AND capture_id IS NULL
  ```

**Version 1 -> 2:**
- Create `memory_tags` junction table:
  ```sql
  CREATE TABLE IF NOT EXISTS memory_tags (
      memory_id TEXT NOT NULL REFERENCES memories(id) ON DELETE CASCADE,
      tag TEXT NOT NULL,
      PRIMARY KEY (memory_id, tag)
  );
  CREATE INDEX IF NOT EXISTS idx_memory_tags_tag ON memory_tags(tag);
  ```
- Backfill from existing JSON tags column:
  ```python
  rows = db.execute("SELECT id, tags FROM memories WHERE tags != '[]'").fetchall()
  for row in rows:
      tags = json.loads(row["tags"]) if isinstance(row["tags"], str) else row["tags"]
      for tag in tags:
          db.execute("INSERT OR IGNORE INTO memory_tags (memory_id, tag) VALUES (?, ?)", (row["id"], tag))
  ```
- Add trigger to keep memory_tags in sync on INSERT:
  ```sql
  CREATE TRIGGER IF NOT EXISTS memories_tags_ai AFTER INSERT ON memories
  BEGIN
      -- Parse JSON tags array and insert into junction table
      -- Use json_each to extract individual tags
      INSERT OR IGNORE INTO memory_tags (memory_id, tag)
      SELECT NEW.id, je.value FROM json_each(NEW.tags) AS je
      WHERE typeof(je.value) = 'text';
  END;
  ```
- Add trigger to sync on UPDATE (delete old, insert new):
  ```sql
  CREATE TRIGGER IF NOT EXISTS memories_tags_au AFTER UPDATE OF tags ON memories
  BEGIN
      DELETE FROM memory_tags WHERE memory_id = OLD.id;
      INSERT OR IGNORE INTO memory_tags (memory_id, tag)
      SELECT NEW.id, je.value FROM json_each(NEW.tags) AS je
      WHERE typeof(je.value) = 'text';
  END;
  ```
- Add trigger to sync on DELETE:
  ```sql
  CREATE TRIGGER IF NOT EXISTS memories_tags_ad AFTER DELETE ON memories
  BEGIN
      DELETE FROM memory_tags WHERE memory_id = OLD.id;
  END;
  ```

3. After each migration step, set user_version: `db.execute(f"PRAGMA user_version = {new_version}")`
4. Commit after all migrations complete.

Call `_migrate_schema(db)` at the end of `_ensure_schema(db)`, after the existing executescript and vec table creation.

Use `ALTER TABLE ... ADD COLUMN` wrapped in try/except to handle the case where column already exists (idempotent). Each migration must be individually guarded so re-running is safe.

Add `_migrate_schema` to `__all__`.

Important: Do NOT remove the existing JSON tags column from memories. The junction table is additive. The JSON column remains for backward compatibility and for `_row_to_dict` deserialization.

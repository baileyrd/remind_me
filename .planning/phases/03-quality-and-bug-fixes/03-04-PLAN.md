  <action>
Audit every `except` block in the codebase and apply the following rules:

**db.py:**

1. `_get_db()` — The sqlite-vec loading block uses `except (ImportError, Exception) as e`. Change to `except ImportError as e` for the import failure case, and add a separate `except sqlite3.OperationalError as e` for the load_extension failure case. Log both at `log.debug` (existing behavior, this is optional functionality).

2. `_embed_and_store()` — Currently `except Exception as e: log.debug(...)`. Change to:
   - `except sqlite3.OperationalError as e` for DB errors (missing vec table)
   - `except (ValueError, TypeError) as e` for embedding computation errors
   - Both should log at `log.warning` (not debug — failed embedding is worth noting)
   - Return False in both cases (existing behavior)

3. `_semantic_search()` — Currently `except Exception as e: log.debug(...)`. Change to:
   - `except sqlite3.OperationalError as e` for DB/vec table errors
   - `except (ValueError, TypeError) as e` for embedding errors
   - Log at `log.warning`
   - Return empty list (existing behavior)

4. `_ensure_schema()` — The vec table creation `except Exception: pass`. Change to `except sqlite3.OperationalError as e: log.debug("sqlite-vec virtual table not available: %s", e)` — log it, don't silently pass.

5. `_row_to_dict()` — `except json.JSONDecodeError: pass`. This is fine as specific, but add `log.debug("Malformed JSON in field %s: %s", key, d[key])`.

**importer.py:**

6. `import_chat_file()` — In the JSONL parsing loop, `except json.JSONDecodeError: continue`. Add `log.debug("Skipping malformed JSONL line")` before continue.

7. `memory_import_directory()` (in tools.py) — `except Exception as e:` around individual file imports. Change to:
   - `except (json.JSONDecodeError, UnicodeDecodeError, FileNotFoundError, OSError) as e` — these are the realistic failure modes
   - Log at `log.warning("Failed to import %s: %s", f.name, e)`

**tools.py:**

8. `memory_search()` — `except sqlite3.OperationalError: pass` for FTS query. This is already specific but add logging: `log.warning("FTS5 query syntax error for query %r: %s", params.query, e)` and include `as e`.

9. `remind_me_reindex()` — The `except Exception: pass` around `vec_rows = db.execute(...)`. Change to `except sqlite3.OperationalError as e: log.debug("memories_vec table not available: %s", e)`.

10. `remind_me_reindex()` — The `except Exception as e: log.debug(...)` in the embedding loop. Change to `except (sqlite3.OperationalError, ValueError, TypeError) as e: log.warning(...)`.

**api.py:**

11. `api_add`, `api_update`, `api_import` — `except Exception: return _json_err("Invalid JSON body")`. Change to `except (json.JSONDecodeError, TypeError, ValueError) as e:` and include the error detail in response: `return _json_err(f"Invalid JSON body: {e}")`.

12. `api_import` — The outer `except Exception as e:` around import processing. Change to `except (FileNotFoundError, OSError, json.JSONDecodeError, UnicodeDecodeError) as e:` with `log.error("Import failed for %s: %s", file_path, e)`.

13. `api_stats` — The tag parsing `except (json.JSONDecodeError, TypeError): pass`. Add `log.debug(...)`.

**Rule: Every `except` block must either:**
- Log the exception at an appropriate level (debug for expected/optional, warning for unexpected but handled, error for failures), OR
- Re-raise it

**No bare `except:` or `except Exception:` should remain unless the failure mode is truly unknown (document why with a comment).**

<task type="auto">
  <name>Task 2: Add user-facing error messages to tool handlers and test error paths</name>
  <files>remind_me_mcp/tools.py, tests/test_tools.py, tests/test_api.py</files>
  <action>
**tools.py — Wrap tool handlers with try/except for user-facing errors:**

Add top-level error handling to each tool handler that could fail from database or I/O errors. Each handler should catch realistic exceptions and return a clear, actionable message:

1. `memory_add` — Wrap DB insert in try/except:
   ```python
   try:
       db.execute(...)
       db.commit()
   except sqlite3.IntegrityError as e:
       log.error("Failed to add memory: %s", e)
       return f"Error: Could not add memory — a memory with this content may already exist."
   except sqlite3.OperationalError as e:
       log.error("Database error adding memory: %s", e)
       return f"Error: Database operation failed — {e}"
   ```

2. `memory_import_chat` — Wrap the import call:
   ```python
   try:
       result = import_chat_file(...)
   except FileNotFoundError:
       return json.dumps({"status": "error", "error": f"File not found: {params.file_path}"})
   except (json.JSONDecodeError, UnicodeDecodeError) as e:
       log.error("Import parse error for %s: %s", params.file_path, e)
       return json.dumps({"status": "error", "error": f"Failed to parse file: {e}"})
   ```

3. `memory_stats` — Wrap DB_PATH.stat() which can fail:
   ```python
   try:
       db_size = round(DB_PATH.stat().st_size / 1_048_576, 2) if DB_PATH.exists() else 0
   except OSError as e:
       log.warning("Could not stat DB file: %s", e)
       db_size = 0
   ```

4. `remind_me_auto_capture` — Wrap the entire DB transaction:
   ```python
   try:
       # ... all INSERTs and UPDATE ...
       db.commit()
   except sqlite3.OperationalError as e:
       log.error("Failed to capture conversation: %s", e)
       return f"Error: Could not capture conversation — database error: {e}"
   ```

**__init__.py** — Package initialization that makes existing entry points work:
```python
"""Remind Me MCP Server — persistent, searchable memory across Claude interfaces."""
from remind_me_mcp.server import mcp  # noqa: F401 — re-export for entry point

# Import tools module to trigger @mcp.tool() registration side effects.
# Without this import, the mcp instance has no tools registered.
import remind_me_mcp.tools  # noqa: F401

__all__ = ["mcp"]
```

This is critical: the `pyproject.toml` entry point `remind_me_mcp:mcp.run` calls `mcp.run()`. For `mcp` to have tools, `tools.py` must have been imported (its `@mcp.tool` decorators run at import time). The `__init__.py` import of `tools` ensures this happens when `remind_me_mcp` is first imported.

**__main__.py** — Extract from `remind_me_mcp.py` lines 2440-2496 (the `if __name__ == "__main__"` block):
- Move the entire CLI logic into a `main()` function
- `import argparse`, `import atexit`, `import signal`, `import sys`, `import logging`
- Import from package: `from remind_me_mcp.config import SERVE_UI, UI_PORT`, `from remind_me_mcp.pid import _read_pid_file, _write_pid_file, _remove_pid_file, _check_ui_server_health, get_server_status`, `from remind_me_mcp.server import mcp`, `from remind_me_mcp.api import _build_api_app`
- `import remind_me_mcp.tools` to ensure tools are registered before `mcp.run()` (belt-and-suspenders with `__init__.py`)
- At bottom of file: `if __name__ == "__main__": main()`
- Module logger: `log = logging.getLogger("remind_me_mcp.__main__")`
- Define `__all__ = ["main"]`

The `main()` function contains the argparse setup, status check mode, UI server mode, and MCP stdio mode — all extracted exactly from the monolith's `__main__` block.

- Change `[project.scripts]` from `remind-me-mcp = "remind_me_mcp:mcp.run"` — keep this same entry point. It works because `__init__.py` re-exports `mcp` from `server.py` and importing `__init__.py` triggers tool registration.
- Add `[tool.hatch.build.targets.wheel]` with `packages = ["remind_me_mcp"]` so hatchling finds the package directory (since it's now a package, not a single .py file at root).

Actually, to prevent import ambiguity, rename the old monolith file: `remind_me_mcp.py` → `remind_me_mcp_ORIGINAL.py` (or add it to `.gitignore`). **[tool.ruff]** — Linting and formatting:
```toml
[tool.ruff]
target-version = "py311"
line-length = 120

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "UP", "B", "SIM", "TCH"]
ignore = [
    "E501",    # line too long (handled by formatter, and we set 120)
]

[tool.ruff.lint.isort]
known-first-party = ["remind_me_mcp"]
```

Explanation of rule selections:
- E, F, W: pycodestyle errors, pyflakes, warnings (core linting)
- I: isort (import sorting)
- N: pep8-naming
- UP: pyupgrade (modern Python patterns)
- B: flake8-bugbear (common bugs)
- SIM: flake8-simplify
- TCH: flake8-type-checking (move type-only imports to TYPE_CHECKING block)

Do NOT enable ASYNC rules yet — the ruff ASYNC rule codes may have changed since August 2025 (noted in STATE.md blockers). Phase 3 will handle async linting after verification.

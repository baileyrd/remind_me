  <behavior>

    - Test 2: compute_vitality decreases over time (days_since=30 returns lower value than days_since=1)
    - Test 3: compute_vitality increases with more accesses (access_count=10 returns higher than access_count=1, same days_since)
    - Test 4: compute_vitality respects base_weight (base_weight=2.0 returns double)
    - Test 5: Bridge protection: get_effective_decay_rate halves decay_rate when access_count >= BRIDGE_THRESHOLD (10)
    - Test 6: Bridge protection: get_effective_decay_rate returns original decay_rate when access_count < BRIDGE_THRESHOLD
    - Test 7: DECAY_RATES dict maps memory types to specific rates
    - Test 8: record_access updates accessed_at, increments access_count, recomputes vitality in database
    - Test 9: record_access applies bridge protection for high-access memories
    - Test 10: is_dormant returns True when vitality < VITALITY_FLOOR (0.05)
  </behavior>
  <action>
Create remind_me_mcp/vitality.py as a focused module:

1. Constants:
   - VITALITY_FLOOR = 0.05
   - BRIDGE_THRESHOLD = 10 (access_count threshold for bridge protection)
   - BRIDGE_MULTIPLIER = 0.5
   - DECAY_RATES: dict mapping memory_type to float:
     - "decision": 0.02 (decisions persist)
     - "preference": 0.03 (preferences are stable)
     - "fact": 0.05 (facts are durable)
     - "insight": 0.07
     - "learning": 0.08
     - "blocker": 0.15 (blockers become stale fast)
     - "action_item": 0.20 (action items decay fastest)
     - "unclassified": 0.10 (default)

2. compute_vitality(base_weight, access_count, decay_rate, days_since_last_access) -> float:
   Pure function. ACT-R formula: base_weight * (access_count + 1) ** 0.5 * math.exp(-decay_rate * days_since_last_access)

3. get_effective_decay_rate(decay_rate, access_count) -> float:
   If access_count >= BRIDGE_THRESHOLD, return decay_rate * BRIDGE_MULTIPLIER. Otherwise return decay_rate.

4. is_dormant(vitality) -> bool:
   Return vitality < VITALITY_FLOOR

5. record_access(memory_id: str) -> float | None:
   - Get DB connection via _get_db()
   - Fetch memory row (accessed_at, access_count, decay_rate, base_weight)
   - If not found, return None
   - Increment access_count
   - Set accessed_at to now
   - Apply bridge protection via get_effective_decay_rate
   - Recompute vitality via compute_vitality (days_since = 0 since we just accessed it)
   - Determine status: 'dormant' if is_dormant(vitality) else 'active'
   - UPDATE the row with new accessed_at, access_count, vitality, status
   - db.commit()
   - Return the new vitality value

All functions have type hints and docstrings. Module has module-level docstring.

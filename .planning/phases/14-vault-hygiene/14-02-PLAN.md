  <action>
Add the remind_me_consolidate tool handler to tools.py following the existing pattern (see remind_me_decompose or remind_me_reclassify for reference):

1. Add imports at top of tools.py:
   - `from remind_me_mcp.consolidation import find_clusters, pick_canonical, merge_cluster`
   - `from remind_me_mcp.models import ConsolidateInput` (add to existing import block)
   - `from remind_me_mcp.vitality import compute_vitality` (add to existing import if not already there)

2. Register the tool with `@mcp.tool()` decorator:
```python
@mcp.tool()
async def remind_me_consolidate(params: ConsolidateInput) -> str:
```

3. Tool handler logic:
   a. Get DB connection via `_get_db()`
   b. Build SQL query to fetch active, non-superseded memories with embeddings:
      ```sql
      SELECT m.id, m.content, m.vitality, m.access_count, m.accessed_at,
             m.category, m.tags, m.memory_type, m.decay_rate, m.base_weight,
             mv.embedding
      FROM memories m
      JOIN memories_vec mv ON mv.rowid = m.rowid
      WHERE m.status = 'active'
        AND m.superseded_by IS NULL
      ```
   c. Add `AND m.category = ?` filter if params.category is not None
   d. Add `LIMIT ?` using params.limit
   e. Build embeddings dict: `{row["id"]: bytes(row["embedding"]) for row in rows}`
   f. Build memories list of dicts from rows (parse tags JSON)
   g. Call `find_clusters(memories, embeddings, params.similarity_threshold)`
   h. If no clusters found, return JSON: `{"clusters_found": 0, "message": "No similar memories found above threshold"}`

4. Dry run mode (params.dry_run is True):
   - For each cluster, call pick_canonical to identify the canonical
   - Build response JSON with cluster details (canonical id/content/vitality, members with id/content/vitality/similarity)
   - To compute similarity for display: use numpy dot product between canonical embedding and each member embedding
   - Return the dry_run response format from RESEARCH.md

5. Auto-merge mode (params.dry_run is False):
   - For each cluster:
     a. Call pick_canonical(cluster)
     b. Call merge_cluster(canonical, members)
     c. UPDATE canonical: content = merged_content, access_count = total_access_count, tags = merged_tags (JSON), updated_at = now
     d. Recalculate vitality: call compute_vitality with canonical's base_weight, new total_access_count, effective decay_rate, days_since=0
     e. UPDATE canonical: vitality = new_vitality, status = 'active'
     f. For each superseded member: UPDATE memories SET superseded_by = canonical_id, updated_at = now
     g. Re-embed canonical: `asyncio.create_task(asyncio.to_thread(_embed_and_store, canonical["id"], merged_content))`
   - Commit all changes in one transaction: `db.commit()`
   - Return JSON with merge results (clusters_merged, memories_superseded, canonical IDs)

6. Update the module docstring at the top of tools.py to reflect the new tool count (19 -> 20 tools).

Key details:
- Wrap merge operations in a single transaction for atomicity
- Use fire-and-forget asyncio.create_task for re-embedding (matching decompose tool pattern)
- Parse tags from JSON string in DB rows before passing to merge_cluster
- Return results as JSON string via json.dumps with indent=2

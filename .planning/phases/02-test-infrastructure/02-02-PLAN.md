- `test_chunk_text_splits_at_newline` ‚Äî text with `\n` but no double newline breaks at newline
- `test_chunk_text_splits_at_sentence` ‚Äî text with `. ` breaks at sentence
- `test_chunk_text_hard_split` ‚Äî text with no natural boundary hard-splits at max_len
- `test_chunk_text_all_chunks_nonempty` ‚Äî no empty strings in output
- `test_chunk_text_preserves_all_content` ‚Äî join of chunks reconstructed roughly equals original (no data loss, allow stripped whitespace)

`_extract_messages_from_json` tests:
- `test_extract_claude_export_format` ‚Äî dict with `chat_messages` containing `content` as list of `{type: "text", text: ...}` blocks extracts correctly
- `test_extract_standard_role_content_list` ‚Äî list of `{role, content}` dicts
- `test_extract_dict_with_messages_key` ‚Äî dict with `"messages"` key delegates correctly
- `test_extract_list_of_conversations` ‚Äî list of dicts each with `"chat_messages"` key
- `test_extract_empty_content_skipped` ‚Äî messages with empty/whitespace content are excluded
- `test_extract_string_content` ‚Äî content as plain string (not list) works

`_filter_messages` tests:
- `test_filter_assistant_messages` ‚Äî mode `"assistant_messages"` returns only assistant/bot content
- `test_filter_user_messages` ‚Äî mode `"user_messages"` returns only user/human content
- `test_filter_all_messages` ‚Äî mode `"all_messages"` includes role prefix `[role] content`
- `test_filter_conversations` ‚Äî mode `"conversations"` joins all messages into one string
- `test_filter_summaries` ‚Äî mode `"summaries"` returns only messages where role contains "summary"
- `test_filter_default` ‚Äî unknown mode returns all content without prefix

`_parse_markdown_chat` tests:
- `test_parse_markdown_with_headers` ‚Äî `## Human` / `## Assistant` format parsed correctly
- `test_parse_markdown_bold_prefix` ‚Äî `**User:**` / `**Assistant:**` format parsed
- `test_parse_markdown_no_structure` ‚Äî plain text with no role markers returned as single entry
- `test_parse_markdown_empty` ‚Äî empty string returns empty list

`_file_hash` tests:
- `test_file_hash_deterministic(tmp_path)` ‚Äî same file content produces same hash
- `test_file_hash_different_content(tmp_path)` ‚Äî different content produces different hash
- `test_file_hash_returns_16_chars(tmp_path)` ‚Äî hash is always 16 hex characters

All tests are synchronous. Use direct imports: `from remind_me_mcp.importer import _chunk_text, ...` and `from remind_me_mcp.formatting import _fmt_memory_md, _fmt_memories`.

<task type="auto">
  <name>Task 1: Integration tests for CRUD tool handlers (add, get, search, list, update, delete)</name>
  <files>tests/test_tools.py</files>
  <action>
Create `tests/test_tools.py`. All tests are `async def` and use the `db_conn` fixture (which monkeypatches `_get_db`).

IMPORTANT: Importing `remind_me_mcp.tools` triggers `@mcp.tool` registration as a side effect. This is fine — it registers tools on the FastMCP instance. Import the handler functions directly: `from remind_me_mcp.tools import memory_add, memory_search, memory_list, memory_get, memory_update, memory_delete, ...`

Also import model classes: `from remind_me_mcp.models import MemoryAddInput, MemorySearchInput, MemoryListInput, MemoryUpdateInput, MemoryDeleteInput, ...`

**memory_add tests:**
- `test_memory_add_basic(db_conn)` — Create `MemoryAddInput(content="Remember to buy milk", category="reminder", tags=["shopping"])`. Await `memory_add(params)`. Assert return string contains "Memory stored" and an ID. Query `db_conn` to verify the row exists with correct content, category, and tags.
- `test_memory_add_stores_metadata(db_conn)` — Add with `metadata={"key": "val"}`. Verify metadata is stored as JSON and deserializes correctly.
- `test_memory_add_default_category(db_conn)` — Add with only content. Verify category defaults to "general".

**memory_get tests:**
- `test_memory_get_exists(db_conn, memory_factory)` — Factory creates a memory. Call `memory_get(memory_id)`. Assert return contains the content and ID.
- `test_memory_get_not_found(db_conn)` — Call with nonexistent ID. Assert return contains "not found".

**memory_search tests:**
- `test_memory_search_fts_match(db_conn)` — Add a memory with content "Python asyncio tutorial". Search for "asyncio". Assert result contains the memory. (FTS5 keyword search, TEST-06.)
- `test_memory_search_no_results(db_conn)` — Search for "zzzznonexistent". Assert "_No memories found._" or empty results.
- `test_memory_search_with_category_filter(db_conn)` — Add two memories with different categories. Search with category filter. Assert only matching category returned.
- `test_memory_search_json_format(db_conn)` — Search with `response_format=ResponseFormat.JSON`. Assert valid JSON returned with "count" and "memories" keys.

**memory_list tests:**
- `test_memory_list_empty(db_conn)` — No memories. Assert returns "_No memories found._".
- `test_memory_list_with_memories(db_conn, memory_factory)` — Factory creates 3 memories. List all. Assert 3 results.
- `test_memory_list_category_filter(db_conn, memory_factory)` — Create memories in different categories. Filter by one category. Assert only matching returned.
- `test_memory_list_pagination(db_conn, memory_factory)` — Create 5 memories. List with limit=2, offset=0. Assert 2 returned. List with limit=2, offset=2. Assert 2 returned.

**memory_update tests:**
- `test_memory_update_content(db_conn, memory_factory)` — Create a memory, update its content. Call `memory_get` to verify new content. Assert return contains "updated".
- `test_memory_update_category_and_tags(db_conn, memory_factory)` — Update category and tags. Verify changes persisted.
- `test_memory_update_not_found(db_conn)` — Update nonexistent ID. Assert return contains "not found".
- `test_memory_update_no_fields(db_conn, memory_factory)` — Call update with only memory_id (no fields to change). Assert "Nothing to update".

**memory_delete tests:**
- `test_memory_delete_exists(db_conn, memory_factory)` — Create a memory, delete it. Assert return contains "deleted". Verify `memory_get` returns "not found".
- `test_memory_delete_not_found(db_conn)` — Delete nonexistent ID. Assert return contains "not found".

**Full CRUD cycle test:**
- `test_crud_cycle(db_conn)` — Add a memory -> get it -> search for it -> update content -> get again (verify new content) -> delete -> get (not found). This is the end-to-end round-trip test.

<task type="auto">
  <name>Task 2: Integration tests for import, auto-capture, reindex, status, and resource handlers</name>
  <files>tests/test_tools.py</files>
  <action>
Append the following tests to `tests/test_tools.py`:

**memory_import_chat tests:**
- `test_import_chat_json(db_conn, sample_chat_json)` — Import the sample JSON file. Assert return JSON contains `"status": "ok"`. Assert `memories_created >= 1`. Query `db_conn` to verify memories exist in the database.
- `test_import_chat_markdown(db_conn, sample_chat_md)` — Import the sample Markdown file. Assert status ok and memories created.
- `test_import_chat_dedup(db_conn, sample_chat_json)` — Import the same file twice. Second import should return `"status": "skipped"`.
- `test_import_chat_search_round_trip(db_conn, sample_chat_json)` — Import a chat file, then search for a keyword from its content ("Python"). Assert at least one result found. This is the end-to-end requirement from Phase 2 success criteria.

**memory_import_directory tests:**
- `test_import_directory(db_conn, tmp_path)` — Create 2 .json files in tmp_path with chat content. Import the directory. Assert `imported >= 2` and `total_memories_created > 0`.
- `test_import_directory_empty(db_conn, tmp_path)` — Empty directory. Assert `files_processed == 0`.

**memory_stats tests:**
- `test_stats_empty(db_conn)` — No memories. Assert total_memories == 0 in result.
- `test_stats_with_data(db_conn, memory_factory)` — Create several memories. Assert stats include correct total and category counts.
- `test_stats_json_format(db_conn, memory_factory)` — Request JSON format. Assert valid JSON with "total_memories" key.

**remind_me_auto_capture tests:**
- `test_auto_capture_creates_two_memories(db_conn)` — Call with conversation and summary text. Assert return contains "captured". Query DB — 2 rows should exist (dialog + summary). Verify one has category "dialog" and the other has the specified category.
- `test_auto_capture_links_via_capture_id(db_conn)` — Capture, then extract the capture_id from the return string. Query both memories and verify their metadata contains matching `capture_id` values, and `linked_dialog`/`linked_summary` cross-reference each other's IDs.
- `test_auto_capture_with_tags(db_conn)` — Capture with tags=["python", "test"]. Verify both memories have the tags.

**remind_me_get_capture tests:**
- `test_get_capture_retrieves_pair(db_conn)` — Auto-capture first, extract capture_id, then call `remind_me_get_capture(capture_id)`. Assert result contains both "Summary" and "Dialog" sections.
- `test_get_capture_not_found(db_conn)` — Call with fake capture_id. Assert "No capture found".

**remind_me_reindex tests:**
- `test_reindex_no_embedder(db_conn)` — With mock_embedder NOT applied (embedder unavailable), call reindex. Assert return mentions "not available" or similar error message. For this test, monkeypatch `remind_me_mcp.embeddings._get_embedder` to return None locally.
- `test_reindex_with_embedder(db_conn, mock_embedder, memory_factory)` — Create a memory, call reindex. Assert return contains "Reindex complete" or "already have embeddings". (Since we can't have the real vec table in memory, this test verifies the function runs without crashing. The mock_embedder provides fake vectors but the vec0 table won't exist — expect graceful handling.)

**remind_me_server_status tests:**
- `test_server_status_no_ui(db_conn)` — Monkeypatch `remind_me_mcp.tools.get_server_status` to return `{"ui_server": "stopped", "ui_url": None, "db_path": "/tmp/test/memory.db", "db_exists": True}`. Call `remind_me_server_status()`. Assert return contains "Not running". (We mock get_server_status because PID detection requires a real process.)

**Resource handler tests:**
- `test_resource_stats(db_conn, memory_factory)` — Create a memory. Call `resource_stats()`. Assert JSON returned with "total_memories" >= 1.
- `test_resource_categories(db_conn, memory_factory)` — Create memories in different categories. Call `resource_categories()`. Assert JSON contains category names.

All async. Use `from remind_me_mcp.tools import remind_me_auto_capture, remind_me_get_capture, remind_me_reindex, remind_me_server_status, resource_stats, resource_categories, memory_import_chat, memory_import_directory, memory_stats`.

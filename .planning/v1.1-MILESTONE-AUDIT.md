### Critical: Unawaited Coroutine in REST API Directory Import

**File:** `remind_me_mcp/api.py`, line 348
**Affected:** PERF-02 REST API path (MCP path is correct)

Phase 8-02 converted `import_directory` to `async def`. The MCP tool handler (`tools.py:455`) was correctly updated with `await`. However, `api_import` in `api.py:348` was **not updated** — it calls `import_directory(...)` without `await`.

**Runtime consequence:** `POST /api/import` with a directory path returns a coroutine object, not the import summary. No files are imported. Python emits `RuntimeWarning: coroutine 'import_directory' was never awaited`.

**Why tests missed it:** All 7 import tests use file paths, not directories. The `p.is_dir()` branch is untested via REST API.

**Fix:** Add `await` + companion test for directory import via REST API.
